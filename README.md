# 📊 لوحة التحكم المصغرة | My Awesome Dashboard

مشروع برمي متميز يهدف إلى بناء واجهة لوحة تحكم مصغرة (Dashboard)، تم تطويره كجزء من التكليف الأكاديمي لتطبيق مفاهيم إدارة النسخ والعمل الجماعي باستخدام **Git** ومنصة **GitHub**.

---

## 🚀 أهداف المشروع وعملية التطوير
تم التركيز في هذا المشروع على محاكاة بيئات العمل الاحترافية في الشركات البرمجية من خلال تحقيق الآتي:
* **استراتيجية الفروع (Branching Strategy):** العمل بالتوازي على ميزات مستقلة عبر فروع مخصصة لكل ميزة (`feature/dashboard` و `feature/login`).
* **إدارة النزاعات (Conflict Resolution):** إحداث نزاع دمج (Merge Conflict) حقيقي وعمدي على نفس الأسطر البرمجية، والتعامل معه وحله يدوياً باحترافية.
* **الالتزامات المنظمة (Conventional Commits):** كتابة سجل الالتزامات التاريخي وفق معايير عالمية مفهومة (مثل: `feat:`, `fix:`, `docs:`).

---

## 🛠️ التقنيات المستخدمة (Tech Stack)
* **HTML5:** لبناء الهيكلية الأساسية لصفحات لوحة التحكم وتسجيل الدخول.
* **Git:** لإدارة النسخ وتتبع التعديلات محلياً.
* **GitHub:** لاستضافة المستودع البرمجي وإدارة العمل المشترك عبر الإنترنت.

---

---

## 📈 سجل الالتزامات التاريخي للمشروع (Commit History)
تم تقسيم الـ Commits في المستودع بالتناوب لتوثيق مساهمة كل مطور بالتساوي:
1. `feat: initialize basic HTML structure for dashboard` (رنين)
2. `docs: add CONTRIBUTING.md for team branching strategy` (أسماء)
3. `feat: change main text to dashboard` (رنين)
4. `feat: add login page heading` (أسماء)
5. `fix: resolve merge conflict between login and dashboard text` (رنين)
6. `docs: finalize README configuration and contributors list` (أسماء)
